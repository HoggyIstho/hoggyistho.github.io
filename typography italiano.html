Strumenti che usiamo

Html 5

Quando si parla di HTML5, la prima cosa che dobbiamo sapere è che si tratta dell'ultima versione della tecnologia HTML, le cui iniziali corrispondono a "HyperText Markup Language", che ha il seguente significato:

  • HyperText, il cui significato è ipertesto, che altro non è che un testo che si collega ad altri contenuti, che possono essere un altro testo o un altro file. Questa è la base del funzionamento del web come lo conosciamo, che altro non è che pagine e risorse interconnesse.
  • Markup, che significa mark o tag, poiché tutte le pagine web sono costruite in base ai tag, dalle prime versioni agli ultimi tag HTML5. Un esempio di tag HTML è quello che identifica un paragrafo, che è composto dal tag, dal contenuto del tag e dalla chiusura del paragrafo.
  • Lingua, il cui significato è linguaggio, perché l'HTML è un linguaggio, cioè ha le sue regole, ha la sua struttura e una serie di convenzioni che ci aiutano a definire sia la struttura che il contenuto di un sito web.

Una cosa importante da tenere a mente e con la quale non bisogna confondersi, è che poiché l'HTML è un linguaggio non significa che sia un linguaggio di programmazione. L'HTML non lo è, in quanto non ha strutture del linguaggio di programmazione come cicli, condizioni, funzioni e così via.

Css

CSS (Cascading Style Sheets) è ciò che viene chiamato un linguaggio di fogli di stile a cascata e viene utilizzato per definire gli elementi scritti in un linguaggio di markup come HTML. I CSS separano il contenuto dalla rappresentazione visiva del sito.

CSS è stato sviluppato dal W3C (World Wide Web Consortium) nel 1996 per un motivo molto semplice. L'HTML non è stato progettato per avere tag per aiutare a formattare la pagina. È fatto solo per scrivere il markup per il sito.

Tag come font sono stati inclusi nella versione HTML 3.2 e questo ha causato molti problemi agli sviluppatori. Poiché i siti Web avevano caratteri, sfondi di colore e stili diversi, il processo di riscrittura del codice è stato lungo, doloroso e costoso. Quindi, i CSS sono stati creati dal W3C per risolvere questo problema.

The relationship between HTML and CSS is very strong. Since HTML is a markup language (that is, it forms the foundation of a site) and CSS emphasizes style (the entire aesthetic part of a website), they go hand in hand.

I CSS non sono tecnicamente una necessità, ma non vuoi avere un sito solo HTML perché sembrerebbe completamente nudo.

JavaScript

Javascript è un linguaggio che può essere utilizzato da professionisti e da coloro che sono nuovi nello sviluppo e nella progettazione di siti web. Non richiede compilazione poiché il linguaggio funziona lato client, i browser sono responsabili dell'interpretazione di questi codici.

Molti confondono Javascript con Java, ma entrambi i linguaggi sono diversi e hanno le loro caratteristiche uniche. Javascript ha il vantaggio di essere incorporato in qualsiasi pagina web, può essere eseguito senza la necessità di installare un altro programma per essere visualizzato.

Java, for its part, has the main characteristic of being a platform-independent language. You can create all kinds of programs that can be run on any computer on the market: Linux, Windows, Apple, etc. Due to its characteristics it is also widely used for the internet.

In sintesi, si può dire che Javascript è un linguaggio interpretato, basato su prototipi, mentre Java è un linguaggio più orientato agli oggetti.

Bootstrap

Bootstrap è un framework front-end utilizzato per sviluppare applicazioni web e siti mobile first, ovvero con un layout che si adatta allo schermo del dispositivo utilizzato dall'utente. Scopri tutto su questa tecnologia e su come usarla!

Bootstrap è un framework CSS utilizzato nelle applicazioni front-end, ovvero nella schermata dell'interfaccia utente, per sviluppare applicazioni che si adattano a qualsiasi dispositivo.

In WordPress, ad esempio, può essere installato come tema o utilizzato per lo sviluppo di plugin o anche al loro interno per stilizzare le loro funzioni. Lo scopo del framework è offrire all'utente un'esperienza più piacevole durante la navigazione in un sito.

Per questo dispone di diverse risorse per configurare gli stili degli elementi della pagina in modo semplice ed efficiente, oltre a facilitare la costruzione di pagine che, allo stesso tempo, si adattano al web e ai dispositivi mobili.

Quanto sopra mostra perché è importante conoscere una potenziale struttura di questo tipo.

GitHub

GitHub è una piattaforma di hosting, di proprietà di Microsoft, che offre agli sviluppatori la possibilità di creare repository di codice e salvarli nel cloud in modo sicuro, utilizzando un sistema di controllo della versione, chiamato Git.

Facilita l'organizzazione del progetto e consente la collaborazione tra più sviluppatori in tempo reale. In altre parole, ci permette di centralizzare il contenuto del repository per collaborare con gli altri membri della nostra organizzazione.

GitHub è basato sul sistema di controllo della versione distribuito di Git, quindi si può fare affidamento sulle sue funzioni e strumenti, sebbene GitHub offra diverse opzioni aggiuntive e la sua interfaccia sia molto più facile da usare, quindi non è assolutamente necessario che le persone che lo usano abbiano grandi conoscenze tecniche.

Tieni d'occhio i nuovi strumenti che utilizzeremo in futuro verranno caricati.
